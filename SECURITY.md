# Security Policy

## Supported Versions

Security fixes are currently applied to the latest release tag.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use one of these methods:

1. GitHub Security Advisories (preferred)
2. Private contact with the maintainer

Include:

- A clear description of the issue
- Steps to reproduce
- Affected version/tag
- Impact assessment

You will receive an acknowledgement as soon as possible, followed by status updates and a coordinated fix/release process.
